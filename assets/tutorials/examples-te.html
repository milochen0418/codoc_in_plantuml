Sequence / BASICS / Simple Message

ఈ ఉదాహరణలో “A నుంచి B కి సందేశం” చూపిస్తుంది.

ఎందుకు ఉపయోగించాలి

  • సందేశాల క్రమాన్ని స్పష్టంగా చూపిస్తుంది.
  • “ఎవరు మొదట మాట్లాడారు” అనేది సులభంగా అర్థమవుతుంది.

ఎలా ఉపయోగించాలి

  1. @startuml తో ప్రారంభించి @enduml తో ముగించండి.
  2. Alice -> Bob: Message లాగా వ్రాయండి.

ఉదాహరణ వినియోగాలు

  • లాగిన్ ఫ్లో: User → System.
  • చాట్‌బాట్: User → Bot.

ప్రాక్టీస్ (సులభం)

  1. పరిశ్రమ 1: పేర్లను మార్చి మీ రోజువారీ సందేశంగా రాయండి.
  2. పరిశ్రమ 2: participant లైన్ జోడించి రెండు సందేశాలు రాయండి.
  3. పరిశ్రమ 3: రెండు లైన్లుగా విడగొట్టండి.
  4. పరిశ్రమ 4: మొదట మాట్లాడే వ్యక్తిని మార్చండి.
  5. పరిశ్రమ 5: మూడో పాత్రను జోడించండి.

PlantUML నియమాలు (పూర్తి జాబితా)

  • @startuml తో ప్రారంభించి @enduml తో ముగించాలి.
  • participant తో ముందుగా ప్రకటించవచ్చు.
  • A -> B కాల్, A --> B రిప్లై.
  • కోలోన్ తర్వాత మెసేజ్ రాయండి.
  • సమయం పై నుంచి కిందకు సాగుతుంది.
  • మెసేజ్ టెక్స్ట్ ఐచ్ఛికం.
  • స్పేస్ ఉన్న పేర్లు quotes లో ఉండాలి.

Sequence / BASICS / Return Message

ఇది “అభ్యర్థన–ప్రత్యుత్తరం” చూపిస్తుంది.

ఎందుకు ఉపయోగించాలి

  • రిక్వెస్ట్ మరియు రిస్పాన్స్ క్రమాన్ని స్పష్టంగా చూపిస్తుంది.

ఎలా ఉపయోగించాలి

  1. A -> B రిక్వెస్ట్.
  2. B --> A రిప్లై.

ఉదాహరణ వినియోగాలు

  • API కాల్‌లు.

ప్రాక్టీస్

  1. Client/Server పేర్లుగా మార్చండి.
  2. మెసేజ్‌లను “query/result” గా మార్చండి.
  3. మరొక రిక్వెస్ట్‑రిస్పాన్స్ జోడించండి.
  4. పాత్రను మార్చి మెసేజ్ సరిపెట్టండి.
  5. ప్రత్యుత్తరానికి ముందు self-message జోడించండి.

PlantUML నియమాలు

  • -> మెసేజ్, --> రిప్లైకి సాధారణం.
  • రిస్పాన్స్ టెక్స్ట్ ఐచ్ఛికం.
  • సమయం పై నుండి కిందకు.

ఎందుకు ఉపయోగించాలి

  • లోపలి స్టెప్‌లను చూపడానికి.

ఎలా ఉపయోగించాలి

  1. Alice -> Alice: Process

ఉదాహరణ వినియోగాలు

  • లోపలి లెక్కలు, రీపీట్.

ప్రాక్టీస్

  1. పేరు మీ పేరుగా మార్చండి.
  2. “డేటా చదవు” అని మార్చండి.
  3. ఇంకొక self‑message జోడించండి.
  4. పేరు Robot గా మార్చండి.
  5. start/done self‑messages జోడించండి.

PlantUML నియమాలు

  • సొంత పేరుకే అంబు ఉండవచ్చు.
  • రూపం: Role -> Role: Text.
  • మెసేజ్ టెక్స్ట్ ఐచ్ఛికం.
