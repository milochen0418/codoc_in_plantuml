Sequence / BASICS / Simple Message

Ipinapakita ng halimbawang ito na “Ang Tao A ay nagpapadala ng mensahe kay Tao B,” parang pagpasa ng note.

Bakit gamitin

  • Ipinapakita ang pagkakasunod ng pag-uusap para madaling maintindihan.
  • Maganda para sa mga kuwentong “sino ang unang nagsalita, sino ang sumunod na sumagot.”

Paano gamitin

  1. Isulat ang @startuml at @enduml.
  2. Gamitin ang Alice -> Bob: Message para ipakitang si A ay nakikipag-usap kay B.

Mga posibleng gamit

  • Login flow: User → System → Server.
  • Chatbot: Ikaw ay nagtatanong → Bot ang sumasagot.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang Alice at Bob ng paboritong pangalan, at gawing pang-araw-araw na usapan ang mensahe.
  2. Pagsasanay 2: Magdagdag muna ng linyang participant, tapos palitan ang mensahe ng “tanong muna, saka sagot.”
  3. Pagsasanay 3: Hatiin ang mensahe sa dalawang linya: tanong muna, saka sagot, at paikliin ang isang linya.
  4. Pagsasanay 4: Palitan kung sino ang unang nagsasalita, at baguhin ang teksto.
  5. Pagsasanay 5: Magdagdag ng pangatlong tao na magsasabi ng isang linya, tapos palitan ang isang pangalan.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • Magsimula sa @startuml at magtapos sa @enduml.
  • Maaaring ideklara ang mga participant gamit ang participant o gamitin nang direkta.
  • A -> B ay mensahe, A --> B ay return.
  • Maglagay ng teksto pagkatapos ng colon: A -> B: Hello.
  • Mula itaas pababa ang daloy ng oras.
  • Maraming mensahe ang ibig sabihin ay mas mahabang usapan.
  • Maaaring wala ang teksto ng mensahe: A -> B ay valid.
  • Ang mga pangalang may space ay dapat nasa quotes.

Sequence / BASICS / Return Message

Ipinapakita ng halimbawang ito ang “request at response,” tulad ng tanong at sagot.

Bakit gamitin

  • Malinaw na ipinapakita ang mga hakbang ng request at reply.

Paano gamitin

  1. A -> B ang request.
  2. B --> A ang response.

Mga posibleng gamit

  • API call: Frontend → Backend → pagbabalik ng data.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang mga papel ng Client at Server, at gawing “kuha ng data / balik ng data” ang mensahe.
  2. Pagsasanay 2: Palitan ang request ng “query,” at ang reply ng “query result.”
  3. Pagsasanay 3: Magdagdag ng isa pang request at reply (apat na linya lahat) at palitan ang ikalawang teksto.
  4. Pagsasanay 4: Iba ang mag-reply at ayusin ang teksto.
  5. Pagsasanay 5: Magdagdag ng self-message na “preparing” bago ang request, tapos gawing mas tiyak ang request.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • -> nagpapadala ng mensahe.
  • --> karaniwang nagpapakita ng return.
  • Magdagdag ng teksto para ilarawan ang nilalaman ng reply.
  • Karaniwan ay pares ang request at reply pero hindi required.
  • I-deklara muna ang participants kung gusto ng malinaw na diagram.
  • Mula itaas pababa ang daloy ng oras.

Sequence / BASICS / Self-Message

Ito ay “pakikipag-usap sa sarili,” na nagpapakita ng internal na hakbang.

Bakit gamitin

  • Pinapakita ang internal na gawain ng isang object nang walang iba.

Paano gamitin

  1. Alice -> Alice: Process

Mga posibleng gamit

  • Mga hakbang sa algorithm, internal na kalkulasyon.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang Alice ng pangalan mo (hal. Me) at palitan ang mensahe sa bagay na gusto mong gawin ngayon.
  2. Pagsasanay 2: Palitan ang aksyon ng “basahin ang data,” tapos magdagdag ng Me -> Me: i-check muli.
  3. Pagsasanay 3: Palitan ang mensahe ng “mag-isip,” tapos magdagdag ng Me -> Me: isulat ang mahahalagang punto.
  4. Pagsasanay 4: Palitan ang pangalan ng Robot, tapos magdagdag ng Robot -> Robot: mag-charge.
  5. Pagsasanay 5: Magdagdag ng Me -> Me: start sa itaas at Me -> Me: done sa ibaba.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • Maaaring parehong participant ang magkabilang dulo ng arrow.
  • Basic na format: Role -> Role: Text.
  • Maaaring gamitin ang pangalan kahit walang naunang deklarasyon.
  • Opsyonal ang teksto pagkatapos ng colon.
  • Maraming self-message ang nagpapakita ng maraming internal na hakbang.
  • Maaaring ipares sa activate / deactivate.
  • Mula itaas pababa ang daloy ng oras.

Sequence / PARTICIPANTS / Types

Ipinapakita ng halimbawang ito ang iba’t ibang icon para sa iba’t ibang role, tulad ng tao, system, database.

Bakit gamitin

  • Mas mabilis makikilala ang role dahil sa icon.

Paano gamitin

  1. Gamitin ang actor, boundary, control, database, entity.

Mga posibleng gamit

  • Login flow: User → Web → Logic → Database.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang User ng Student at gawing “login ng homework” ang mensahe.
  2. Pagsasanay 2: Magdagdag ng boundary App, tapos idaan muna ang arrow sa App.
  3. Pagsasanay 3: Palitan ang database DB ng database Books at palitan ang mensahe.
  4. Pagsasanay 4: Magdagdag ng role na control at isang linya ng interaction.
  5. Pagsasanay 5: Ayusin ang pagkakasunod ng participants at i-update ang mga mensahe.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • Ang uri ng role ay nagbabago ng hitsura, hindi ng pangalan.
  • Gamitin ang -> para sa interaction.
  • Format ng deklarasyon: actor/boundary/control/database/entity Name.
  • I-deklara muna ang roles, saka gamitin ang pangalan.
  • Ang hindi nade-deklarang pangalan ay normal na participant.

Sequence / PARTICIPANTS / Colors & Styles

Ipinapakita ng halimbawang ito ang mga kulay at estilo para luminaw ang diagram.

Bakit gamitin

  • I-highlight ang mahahalagang role gamit ang kulay.

Paano gamitin

  1. Gamitin ang skinparam para ayusin ang estilo.
  2. Maglagay ng kulay pagkatapos ng role: participant Alice #AliceBlue.

Mga posibleng gamit

  • Ipakita ang “core system” sa matingkad na kulay.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang Alice at ang kulay nito.
  2. Pagsasanay 2: Palitan ang mensahe ng “start,” at gawing mas madilim ang ibang role.
  3. Pagsasanay 3: Magdagdag ng pangatlong role at ibang kulay.
  4. Pagsasanay 4: Palitan ang skinparam sequenceMessageAlign at ayusin ang isang linya.
  5. Pagsasanay 5: Gumamit ng pangalan ng style (gaya ng #technology) at palitan ang mensahe.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • skinparam ang kumokontrol sa alignment, font, kulay, at iba pa.
  • Maaaring sundan ang role ng kulay o style name.
  • Maaaring pangalan ng kulay o hex, gaya ng #AliceBlue.
  • Kung muling ideklara ang parehong role, ang huli ang masusunod.
  • Hindi binabago ng style settings ang syntax ng mensahe.

Sequence / CONTROL FLOW / Alt/Else (Choices)

Ipinapakita ng halimbawa ang “if…else…,” parang maraming pagpipilian.

Bakit gamitin

  • Ipinapakita ang iba’t ibang sangay ayon sa kundisyon.

Paano gamitin

  1. Gamitin ang alt para magsimula, else para sa ibang sangay, at end para magtapos.

Mga posibleng gamit

  • Tagumpay o kabiguan ng bayad.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang kundisyon ng “may pera / walang pera,” at baguhin ang mensahe.
  2. Pagsasanay 2: Magdagdag ng pangatlong else branch at magsulat ng mensahe.
  3. Pagsasanay 3: Palitan ang pamagat ng alt sa sarili mong senaryo, saka i-edit ang mga mensahe.
  4. Pagsasanay 4: Magdagdag ng isang linya sa bawat branch at palitan ang mga pamagat.
  5. Pagsasanay 5: Gawing “subukan muli pagkatapos mabigo” ang isang branch, at gawing ibang ending ang isa.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • alt Condition ang nagsisimula ng branch.
  • Maaaring magkaroon ng maraming else branches.
  • Tapusin gamit ang end.
  • Bawat branch ay maaaring may maraming mensahe.
  • Opsyonal ang teksto ng kundisyon pero mas mainam kung meron.

Sequence / CONTROL FLOW / Loop

Ipinapakita ng halimbawang ito ang paulit-ulit na gawain.

Bakit gamitin

  • Ipinapakita ang mga hakbang na inuulit.

Paano gamitin

  1. Gamitin ang loop at end para balutin ang paulit-ulit na linya.

Mga posibleng gamit

  • Mag-check ng status bawat segundo.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang label ng loop sa “bawat 3 beses,” at palitan ang mensahe.
  2. Pagsasanay 2: Magdagdag ng isa pang mensahe sa loob ng loop at paikliin ang unang mensahe.
  3. Pagsasanay 3: Palitan ang mga role at gawing “nagche-check” ang mensahe.
  4. Pagsasanay 4: Palitan ang loop label ng kundisyon tulad ng “hanggang matapos,” at ayusin ang mensahe.
  5. Pagsasanay 5: Magdagdag ng start/end messages sa labas ng loop at i-customize.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • loop count/condition maaaring nasa iisang linya.
  • end ang nagtatapos ng loop block.
  • Maaaring may maraming mensahe sa loob ng loop.

Sequence / CONTROL FLOW / Parallel (Par)

Ipinapakita ng halimbawa ang dalawang bagay na nangyayari nang sabay.

Bakit gamitin

  • Ipinapakita ang parallel na gawain.

Paano gamitin

  1. Gamitin ang par para magsimula, else para sa ibang parallel branch, at end para magtapos.

Mga posibleng gamit

  • I-notify ang dalawang serbisyo nang sabay.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang branches ng “send email” at “send SMS,” at palitan ang mga role.
  2. Pagsasanay 2: Palitan ang mga role at magdagdag ng isang linya sa bawat branch.
  3. Pagsasanay 3: Magdagdag ng pangatlong branch (else) at magsulat ng mensahe, tapos paikliin ang isang linya.
  4. Pagsasanay 4: Palitan ang isang branch ng “dahan-dahan gawin,” magdagdag ng end message, at ayusin ang text ng kabila.
  5. Pagsasanay 5: Magdagdag ng “start task” bago ang par at “done” pagkatapos ng end.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • par gumagawa ng parallel block.
  • Bawat branch ay maaaring may sariling sequence ng mensahe.
  • else lumilipat sa ibang parallel branch.
  • Taposin gamit ang end.

Sequence / ADVANCED / Activation

Gumagamit ang halimbawa ng activation bars para ipakitang kailan abala ang isang role.

Bakit gamitin

  • Ipinapakita kung sino ang gumagawa at gaano katagal.

Paano gamitin

  1. Gamitin ang activate Role para magsimula.
  2. Gamitin ang deactivate Role para magtapos.

Mga posibleng gamit

  • Ipakita ang oras ng pagproseso ng server.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang role ng Server at ang mensahe ng “processing.”
  2. Pagsasanay 2: Magdagdag ng request line, tapos ilagay ang activate pagkatapos ng request.
  3. Pagsasanay 3: Magdagdag ng mid-progress message tulad ng “halfway,” tapos palitan ito ng sarili mong text.
  4. Pagsasanay 4: Magdagdag ng ibang role kasama ang activation at mensahe nito.
  5. Pagsasanay 5: Ilipat ang activation start/end ng isang linya pababa at ayusin ang mensahe.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • Maaaring ilagay ang activate/deactivate bago o pagkatapos ng mga mensahe.
  • activate at deactivate ay dapat magkapareha.
  • Maaaring mag-insert ng mensahe habang active ang role.

Sequence / ADVANCED / Notes

Ipinapakita ng halimbawa kung paano magdagdag ng sticky-note annotations.

Bakit gamitin

  • Nagdadagdag ng paliwanag para sa kalinawan.

Paano gamitin

  1. Gamitin ang note left o note right.
  2. Tapusin ang note gamit ang end note.

Mga posibleng gamit

  • Paalala na “maaaring mag-fail ang hakbang na ito.”

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang teksto ng note ng sarili mong paalala.
  2. Pagsasanay 2: Ilipat ang note sa kanan at palitan ang teksto.
  3. Pagsasanay 3: Magdagdag ng pangalawang note sa tabi ng ibang role.
  4. Pagsasanay 4: Gawing dalawang linya ang note at maglagay ng dahilan sa ikalawang linya.
  5. Pagsasanay 5: Ilipat ang note pagkatapos ng ibang mensahe.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • Maaaring ilagay ang notes sa left, right, o over.
  • Ang multi-line notes ay gumagamit ng note at end note.
  • Hindi binabago ng notes ang daloy ng mensahe.

Use Case / BASICS / Simple Use Case

Ang Use case diagram ay listahan ng “mga kaya ng system.”

Bakit gamitin

  • Malinaw na nililista ang mga function na kailangan ng user.

Paano gamitin

  1. Gamitin ang actor para sa user.
  2. Gamitin ang (UseCase) para sa function.

Mga posibleng gamit

  • Login, search, payment, atbp.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang user ng Teacher at ang use case ng “create quiz.”
  2. Pagsasanay 2: Magdagdag ng pangalawang use case para magpakita ng isa pang function.
  3. Pagsasanay 3: Palitan ang pangalan ng use case at actor.
  4. Pagsasanay 4: Magdagdag ng pangalawang actor at ikabit sa isang use case.
  5. Pagsasanay 5: Palitan ang use case ng sarili mong senaryo.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • Ang use case ay nasa parentheses: (Login).
  • Ang actor ay stick figure: actor User.
  • I-konekta ang actor sa use case gamit ang isang linya.

Use Case / RELATIONSHIPS / Include/Extend

Ipinapakita ang inclusion o extension sa pagitan ng use cases.

Bakit gamitin

  • Markahan ang shared steps o optional flows.

Paano gamitin

  1. Gamitin ang <> para sa “laging kasama.”
  2. Gamitin ang <> para sa “opsyonal.”

Mga posibleng gamit

  • Login laging may validation; payment maaaring may discount via extend.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang use cases ng sarili mong halimbawa.
  2. Pagsasanay 2: Ipagpalit ang include at extend.
  3. Pagsasanay 3: Magdagdag ng isa pang use case at i-include ito.
  4. Pagsasanay 4: Baligtarin ang arrow at i-update ang teksto.
  5. Pagsasanay 5: Palitan ang teksto ng relasyon sa sarili mong salita.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • Include syntax: (A) .> (B) : <>.
  • Extend syntax: (A) .> (B) : <>.
  • Gumamit ng dashed arrows.

Class / BASICS / Simple Class

Ang class diagrams ay naglalarawan ng attributes at methods.

Bakit gamitin

  • Ayusin ang data structures.

Paano gamitin

  1. Gamitin ang class para magdeklara ng class.
  2. Ilagay ang attributes at methods sa loob ng braces.

Mga posibleng gamit

  • Disenyo ng data model ng programa.

Pagsasanay (madali para sa mga bata)

  1. Pagsasanay 1: Palitan ang class name ng gusto mo, tulad ng Book.
  2. Pagsasanay 2: Magdagdag ng isang attribute o method.
  3. Pagsasanay 3: Paikliin ang field name o isalin ito.
  4. Pagsasanay 4: Magdagdag ng pangalawang class at dalawang attributes.
  5. Pagsasanay 5: Palitan ang method ng sarili mong aksyon.

Mga tuntunin ng PlantUML (kumpletong listahan)

  • Format ng class: class Name { ... }.
  • Ang attributes ay fields, ang methods ay may ().
  • Pinapayagan ang empty classes.

Activity / BASICS / Flow

Pagsasanay (madali para sa mga bata)
