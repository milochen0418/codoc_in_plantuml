Sequence / BASICS / Simple Message

ตัวอย่างนี้แสดงว่า “คน A ส่งข้อความหนึ่งไปให้คน B” เหมือนส่งกระดาษโน้ต

ทำไมถึงใช้

  • แสดงลำดับการสนทนาให้เข้าใจง่าย
  • เหมาะกับเรื่องเล่าแบบ “ใครพูดก่อน ใครตอบต่อ”

วิธีใช้

  1. เขียน @startuml และ @enduml
  2. ใช้ Alice -> Bob: Message เพื่อแสดงว่า A พูดกับ B

ตัวอย่างการใช้งาน

  • ขั้นตอนล็อกอิน: User → System → Server
  • แชตบอต: คุณถาม → บอตตอบ

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยน Alice และ Bob เป็นชื่อที่ชอบ แล้วเปลี่ยนข้อความเป็นบทสนทนาประจำวัน
  2. แบบฝึกหัด 2: เพิ่มบรรทัด participant ก่อน แล้วเปลี่ยนข้อความเป็น “ถามก่อน แล้วค่อยตอบ”
  3. แบบฝึกหัด 3: แยกข้อความเป็นสองบรรทัด: ถามก่อน แล้วตอบ และย่อหนึ่งบรรทัดให้สั้นลง
  4. แบบฝึกหัด 4: สลับคนที่พูดก่อน แล้วเปลี่ยนข้อความ
  5. แบบฝึกหัด 5: เพิ่มคนที่สามให้พูดหนึ่งบรรทัด แล้วเปลี่ยนชื่อหนึ่งคน

กฎไวยากรณ์ PlantUML (รายการครบถ้วน)

  • เริ่มด้วย @startuml และจบด้วย @enduml
  • ผู้ร่วมสนทนาสามารถประกาศด้วย participant หรือใช้งานตรง ๆ ได้
  • A -> B คือการส่งข้อความ, A --> B คือการตอบกลับ
  • ใส่ข้อความหลังโคลอน: A -> B: Hello
  • เวลาจะแสดงจากบนลงล่าง
  • หลายบรรทัดข้อความคือบทสนทนาที่ยาวขึ้น
  • ข้อความสามารถละได้: A -> B ใช้ได้
  • ชื่อที่มีช่องว่างควรใส่เครื่องหมายคำพูด

Sequence / BASICS / Return Message

ตัวอย่างนี้แสดง “คำขอและคำตอบ” เหมือนถาม‑ตอบ

ทำไมถึงใช้

  • เห็นขั้นตอนคำขอและคำตอบชัดเจน

วิธีใช้

  1. A -> B คือคำขอ
  2. B --> A คือคำตอบ

ตัวอย่างการใช้งาน

  • เรียก API: Frontend → Backend → ส่งข้อมูลกลับ

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยนบทบาทเป็น Client และ Server แล้วเปลี่ยนข้อความเป็น “ขอข้อมูล / ส่งข้อมูลกลับ”
  2. แบบฝึกหัด 2: เปลี่ยนคำขอเป็น “query” และคำตอบเป็น “query result”
  3. แบบฝึกหัด 3: เพิ่มคำขอ/คำตอบอีกชุด (รวม 4 บรรทัด) แล้วเปลี่ยนข้อความบรรทัดที่สอง
  4. แบบฝึกหัด 4: ให้คนอื่นเป็นผู้ตอบ แล้วปรับข้อความ
  5. แบบฝึกหัด 5: เพิ่ม self-message “เตรียมตัว” ก่อนคำขอ แล้วทำคำขอให้เฉพาะเจาะจงขึ้น

กฎไวยากรณ์ PlantUML (รายการครบถ้วน)

  • -> ใช้ส่งข้อความ
  • --> มักใช้แสดงการตอบกลับ
  • เพิ่มข้อความเพื่ออธิบายเนื้อหาคำตอบ
  • คำขอและคำตอบมักมาเป็นคู่ แต่ไม่จำเป็นเสมอไป
  • ประกาศผู้ร่วมสนทนาก่อนจะช่วยให้ชัดเจน
  • เวลาจะไหลจากบนลงล่าง

Sequence / BASICS / Self-Message

นี่คือ “คุยกับตัวเอง” ซึ่งแสดงขั้นตอนภายใน

ทำไมถึงใช้

  • แสดงการทำงานภายในของวัตถุโดยไม่ต้องมีคนอื่น

วิธีใช้

  1. Alice -> Alice: Process

ตัวอย่างการใช้งาน

  • ขั้นตอนอัลกอริทึม การคำนวณภายใน

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยน Alice เป็นชื่อคุณ (เช่น Me) และเปลี่ยนข้อความเป็นสิ่งที่อยากทำวันนี้
  2. แบบฝึกหัด 2: เปลี่ยนการกระทำเป็น “อ่านข้อมูล” แล้วเพิ่ม Me -> Me: ตรวจอีกครั้ง
  3. แบบฝึกหัด 3: เปลี่ยนข้อความเป็น “คิด” แล้วเพิ่ม Me -> Me: เขียนประเด็นสำคัญ
  4. แบบฝึกหัด 4: เปลี่ยนชื่อเป็น Robot แล้วเพิ่ม Robot -> Robot: ชาร์จ
  5. แบบฝึกหัด 5: เพิ่ม Me -> Me: start ไว้ด้านบน และ Me -> Me: done ไว้ด้านล่าง

กฎไวยากรณ์ PlantUML (รายการครบถ้วน)

  • ปลายทั้งสองของลูกศรอาจเป็นคนเดียวกันได้
  • รูปแบบพื้นฐาน: Role -> Role: Text
  • สามารถใช้ชื่อได้โดยไม่ต้องประกาศก่อน
  • ข้อความหลังโคลอนเป็นตัวเลือก
  • หลาย self-message แสดงหลายขั้นตอนภายใน
  • สามารถใช้คู่กับ activate / deactivate
  • เวลาจะไหลจากบนลงล่าง

Sequence / PARTICIPANTS / Types

ตัวอย่างนี้แสดงไอคอนต่าง ๆ สำหรับบทบาทต่างกัน เช่น คน ระบบ ฐานข้อมูล

ทำไมถึงใช้

  • ไอคอนช่วยให้จำบทบาทได้เร็ว

วิธีใช้

  1. ใช้ actor, boundary, control, database, entity

ตัวอย่างการใช้งาน

  • ขั้นตอนล็อกอิน: User → Web → Logic → Database

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยน User เป็น Student และเปลี่ยนข้อความเป็น “ล็อกอินการบ้าน”
  2. แบบฝึกหัด 2: เพิ่ม boundary App แล้วให้ลูกศรไปที่ App ก่อน
  3. แบบฝึกหัด 3: เปลี่ยน database DB เป็น database Books และเปลี่ยนข้อความ
  4. แบบฝึกหัด 4: เพิ่มบทบาท control แล้วเขียนเส้นการโต้ตอบหนึ่งบรรทัด
  5. แบบฝึกหัด 5: สลับลำดับผู้ร่วมสนทนา แล้วปรับข้อความตาม

กฎไวยากรณ์ PlantUML (รายการครบถ้วน)

  • ชนิดบทบาทเปลี่ยนรูปร่าง ไม่เปลี่ยนชื่อ
  • การโต้ตอบใช้ ->
  • รูปแบบประกาศ: actor/boundary/control/database/entity Name
  • ประกาศบทบาทก่อน แล้วค่อยใช้ชื่อ
  • ชื่อที่ไม่ประกาศจะเป็นผู้ร่วมสนทนาทั่วไป

Sequence / PARTICIPANTS / Colors & Styles

ตัวอย่างนี้ใช้สีและสไตล์เพื่อให้แผนภาพชัดเจนขึ้น

ทำไมถึงใช้

  • เน้นบทบาทสำคัญด้วยสีได้

วิธีใช้

  1. ใช้ skinparam เพื่อปรับสไตล์
  2. ใส่สีหลังบทบาท: participant Alice #AliceBlue

ตัวอย่างการใช้งาน

  • เน้น “ระบบหลัก” ด้วยสีสว่าง

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยนชื่อ Alice และเปลี่ยนสีเป็นสีอื่น
  2. แบบฝึกหัด 2: เปลี่ยนข้อความเป็น “เริ่มต้น” และทำอีกบทบาทให้เข้มขึ้น
  3. แบบฝึกหัด 3: เพิ่มบทบาทที่สามและกำหนดสีต่างกัน
  4. แบบฝึกหัด 4: เปลี่ยน skinparam sequenceMessageAlign แล้วปรับข้อความหนึ่งบรรทัด
  5. แบบฝึกหัด 5: ใช้ชื่อสไตล์ (เช่น #technology) กับหนึ่งบทบาท และเปลี่ยนข้อความ

กฎไวยากรณ์ PlantUML (รายการครบถ้วน)

  • skinparam ควบคุมการจัดแนว ฟอนต์ สี ฯลฯ
  • บทบาทสามารถตามด้วยสีหรือชื่อสไตล์ได้
  • สีสามารถเป็นชื่อหรือค่า hex เช่น #AliceBlue
  • ถ้าประกาศบทบาทเดียวกันซ้ำ จะใช้ค่าล่าสุด
  • การตั้งค่าสไตล์ไม่เปลี่ยนไวยากรณ์ของข้อความ

Sequence / CONTROL FLOW / Alt/Else (Choices)

ตัวอย่างนี้แสดง “ถ้า...อย่างอื่น...” เหมือนการเลือกหลายทาง

ทำไมถึงใช้

  • แสดงกิ่งก้านต่าง ๆ ตามเงื่อนไขต่างกัน

วิธีใช้

  1. ใช้ alt เพื่อเริ่ม, else สำหรับกิ่งอื่น และ end เพื่อจบ

ตัวอย่างการใช้งาน

  • การชำระเงินสำเร็จหรือไม่สำเร็จ

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยนเงื่อนไขเป็น “มีเงิน / ไม่มีเงิน” แล้วเปลี่ยนข้อความ
  2. แบบฝึกหัด 2: เพิ่มกิ่ง else ที่สาม และเขียนข้อความหนึ่งบรรทัด
  3. แบบฝึกหัด 3: เปลี่ยนชื่อหัวข้อ alt เป็นสถานการณ์ของคุณ แล้วแก้ไขข้อความ
  4. แบบฝึกหัด 4: เพิ่มหนึ่งบรรทัดในแต่ละกิ่ง และเปลี่ยนชื่อหัวข้อ
  5. แบบฝึกหัด 5: ทำหนึ่งกิ่งเป็น “ลองใหม่หลังล้มเหลว” และอีกกิ่งเป็นตอนจบที่ต่างกัน

กฎไวยากรณ์ PlantUML (รายการครบถ้วน)

  • alt Condition เริ่มกิ่งหนึ่ง
  • สามารถมีหลายกิ่ง else
  • จบด้วย end
  • แต่ละกิ่งมีหลายข้อความได้
  • ข้อความเงื่อนไขเป็นตัวเลือก แต่แนะนำให้ใส่

Sequence / CONTROL FLOW / Loop

ตัวอย่างนี้แสดงการทำงานซ้ำ ๆ

ทำไมถึงใช้

  • แสดงขั้นตอนที่ต้องทำซ้ำ

วิธีใช้

  1. ครอบบรรทัดที่ซ้ำด้วย loop และ end

ตัวอย่างการใช้งาน

  • ตรวจสถานะทุก ๆ วินาที

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยนป้าย loop เป็น “ทุก 3 ครั้ง” แล้วเปลี่ยนข้อความ
  2. แบบฝึกหัด 2: เพิ่มอีกหนึ่งข้อความใน loop แล้วทำให้ข้อความเดิมสั้นลง
  3. แบบฝึกหัด 3: เปลี่ยนชื่อบทบาทและเปลี่ยนข้อความเป็น “กำลังตรวจ”
  4. แบบฝึกหัด 4: เปลี่ยนป้าย loop เป็นเงื่อนไขเช่น “จนกว่าจะเสร็จ” แล้วปรับข้อความ
  5. แบบฝึกหัด 5: เพิ่มข้อความเริ่ม/จบไว้ก่อนและหลัง loop

กฎไวยากรณ์ PlantUML (รายการครบถ้วน)

  • loop count/condition สามารถอยู่บรรทัดเดียวกันได้
  • end จบ block ของ loop
  • หลายข้อความอยู่ใน loop ได้

Sequence / CONTROL FLOW / Parallel (Par)

ตัวอย่างนี้แสดงสองอย่างเกิดพร้อมกัน

ทำไมถึงใช้

  • แสดงงานที่ทำคู่ขนาน

วิธีใช้

  1. ใช้ par เพื่อเริ่ม, else สำหรับกิ่งคู่ขนานอื่น และ end เพื่อจบ

ตัวอย่างการใช้งาน

  • แจ้งสองบริการพร้อมกัน

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยนกิ่งเป็น “ส่งอีเมล” และ “ส่ง SMS” แล้วเปลี่ยนชื่อบทบาท
  2. แบบฝึกหัด 2: เปลี่ยนบทบาทและเพิ่มอีกหนึ่งบรรทัดในแต่ละกิ่ง
  3. แบบฝึกหัด 3: เพิ่มกิ่งที่สาม (else) และเขียนข้อความหนึ่งบรรทัด แล้วทำให้หนึ่งบรรทัดสั้นลง
  4. แบบฝึกหัด 4: เปลี่ยนกิ่งหนึ่งเป็น “ทำช้า ๆ” เพิ่มข้อความจบ แล้วปรับข้อความกิ่งอื่น
  5. แบบฝึกหัด 5: เพิ่มข้อความ “เริ่มงาน” ก่อน par และ “เสร็จแล้ว” หลัง end

กฎไวยากรณ์ PlantUML (รายการครบถ้วน)

  • par สร้าง block แบบคู่ขนาน
  • แต่ละกิ่งมีลำดับข้อความของตัวเองได้
  • else สลับไปกิ่งคู่ขนานอีกกิ่ง
  • จบด้วย end

Sequence / ADVANCED / Activation

ตัวอย่างนี้ใช้แถบ activation เพื่อแสดงว่าใครกำลังทำงานอยู่

ทำไมถึงใช้

  • บอกว่าใครทำงานและนานแค่ไหน

วิธีใช้

  1. ใช้ activate Role เพื่อเริ่ม
  2. ใช้ deactivate Role เพื่อจบ

ตัวอย่างการใช้งาน

  • แสดงเวลาที่เซิร์ฟเวอร์กำลังประมวลผลคำขอ

แบบฝึกหัด (ง่ายสำหรับเด็ก)

  1. แบบฝึกหัด 1: เปลี่ยนบทบาทเป็น Server และเปลี่ยนข้อความเป็น “processing”
  2. แบบฝึกหัด 2: เพิ่มบรรทัดคำขอ แล้วใส่ activate หลังคำขอทันที
  3. แบบฝึกหัด 3: เพิ่มข้อความกลางทาง เช่น “halfway” แล้วแทนด้วยข้อความของคุณ
  4. แบบฝึกหัด 4: เพิ่มอีกบทบาทหนึ่ง พร้อม activation และข้อความของตัวเอง

  • ||--o{ มักหมายถึง one-to-many
