Sequence / BASICS / Simple Message

এই উদাহরণে দেখানো হয় “ব্যক্তি A ব্যক্তি B‑কে একটি বার্তা পাঠায়”, যেমন একটি নোট দেওয়া।

কেন ব্যবহার করবেন

  • কথোপকথনের ক্রম সহজভাবে দেখায়।
  • “কে আগে বলল, কে পরে উত্তর দিল” ধরনের গল্পে দারুণ কাজ করে।

কীভাবে ব্যবহার করবেন

  1. @startuml এবং @enduml লিখুন।
  2. Alice -> Bob: Message দিয়ে A‑এর B‑কে বলা দেখান।

সম্ভাব্য ব্যবহার

  • লগইন ফ্লো: ব্যবহারকারী → সিস্টেম → সার্ভার।
  • চ্যাটবট: তুমি জিজ্ঞেস করো → বট উত্তর দেয়।

অনুশীলন (শিশুদের জন্য সহজ)

  1. অনুশীলন ১: Alice ও Bob‑কে পছন্দের নামে বদলাও, আর বাক্যটা দৈনন্দিন কথোপকথনে বদলাও।
  2. অনুশীলন ২: আগে একটি participant লাইন যোগ করো, তারপর “প্রথমে প্রশ্ন, পরে উত্তর” করো।
  3. অনুশীলন ৩: বার্তাটি দুই লাইনে ভাগ করো—প্রশ্ন তারপর উত্তর—এবং একটি লাইন ছোট করো।
  4. অনুশীলন ৪: কে আগে বলছে তা বদলাও এবং লেখা পরিবর্তন করো।
  5. অনুশীলন ৫: তৃতীয় একজন যোগ করো এবং কারও নাম বদলাও।

PlantUML সিনট্যাক্স নিয়ম (সম্পূর্ণ তালিকা)

  • @startuml দিয়ে শুরু এবং @enduml দিয়ে শেষ।
  • অংশগ্রহণকারী participant দিয়ে ঘোষণা করা যায় বা সরাসরি ব্যবহার করা যায়।
  • A -> B হলো কল, A --> B হলো রিটার্ন।
  • টেক্সট কোলনের পরে লেখা হয়: A -> B: Hello।
  • সময় উপরে থেকে নিচে যায়।
  • বেশি বার্তা মানে দীর্ঘ কথোপকথন।
  • বার্তার টেক্সট ঐচ্ছিক: A -> B বৈধ।
  • ফাঁকা‑সহ নাম হলে কোটেশনে লিখতে হয়।

Sequence / BASICS / Return Message

এই উদাহরণে “অনুরোধ ও উত্তর” দেখানো হয়, যেমন প্রশ্ন‑উত্তর।

কেন ব্যবহার করবেন

  • অনুরোধ ও উত্তরের ধাপগুলো স্পষ্ট দেখায়।

কীভাবে ব্যবহার করবেন

  1. A -> B হলো অনুরোধ।
  2. B --> A হলো উত্তর।

সম্ভাব্য ব্যবহার

  • API কল: ফ্রন্টএন্ড → ব্যাকএন্ড → ডেটা ফেরত।

অনুশীলন (শিশুদের জন্য সহজ)

  1. অনুশীলন ১: ভূমিকা বদলে Client ও Server করো, বার্তা করো “ডেটা চাই / ডেটা দাও”।
  2. অনুশীলন ২: অনুরোধকে “query”, উত্তরকে “query result” করো।
  3. অনুশীলন ৩: আরেক জোড়া অনুরোধ‑উত্তর যোগ করো (৪ লাইন) এবং দ্বিতীয় টেক্সট বদলাও।
  4. অনুশীলন ৪: ভিন্ন কেউ উত্তর দিক এবং টেক্সট মিলিয়ে নাও।
  5. অনুশীলন ৫: অনুরোধের আগে “প্রস্তুতি” self‑message যোগ করো।

PlantUML সিনট্যাক্স নিয়ম (সম্পূর্ণ তালিকা)

  • -> বার্তা পাঠায়।
  • --> সাধারণত রিটার্ন বোঝায়।
  • উত্তরের বিষয় বোঝাতে টেক্সট যোগ করুন।
  • অনুরোধ‑উত্তর সাধারণত জোড়ায় থাকে, তবে বাধ্যতামূলক নয়।
  • পরিষ্কার করতে চাইলে আগে অংশগ্রহণকারী ঘোষণা করুন।
  • সময় উপরে থেকে নিচে যায়।

  1. Alice -> Alice: Process

অনুশীলন (শিশুদের জন্য সহজ)

  1. অনুশীলন ১: Alice‑কে নিজের নামে বদলাও (যেমন Me) এবং বার্তা বদলাও।
  2. অনুশীলন ২: ক্রিয়াটি “ডেটা পড়ো” করো এবং Me -> Me: আবার যাচাই যোগ করো।
  3. অনুশীলন ৩: বার্তাটি “ভাবো” করো এবং Me -> Me: মূল পয়েন্ট লেখো যোগ করো।
  4. অনুশীলন ৪: নাম Robot করো এবং Robot -> Robot: চার্জ যোগ করো।
  5. অনুশীলন ৫: শুরুতে Me -> Me: start এবং শেষে Me -> Me: done যোগ করো।

  • বেসিক ফরম্যাট: Role -> Role: Text।
  • নাম ঘোষণা ছাড়াও ব্যবহার করা যায়।
  • কোলনের পরে টেক্সট ঐচ্ছিক।
  • একাধিক self‑message আরও অভ্যন্তরীণ ধাপ দেখায়।
  • activate / deactivate দিয়ে ব্যবহার করা যায়।
  • সময় উপরে থেকে নিচে যায়।
