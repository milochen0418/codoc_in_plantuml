Sequence / BASICS / Simple Message

Contoh ini menunjukkan “Orang A mengirim pesan ke orang B”, seperti mengirim catatan.

Mengapa digunakan

  • Menunjukkan urutan percakapan secara sederhana.
  • Cocok untuk cerita “siapa bicara dulu, siapa menjawab setelahnya”.

Cara menggunakan

  1. Tulis @startuml dan @enduml.
  2. Gunakan Alice -> Bob: Message untuk menunjukkan A berbicara ke B.

Contoh penggunaan

  • Alur login: User → Sistem → Server.
  • Chatbot: Kamu tanya → Bot menjawab.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ganti Alice dan Bob dengan nama favorit, dan ubah kalimat menjadi percakapan harian.
  2. Latihan 2: Tambahkan baris participant dulu, lalu ubah jadi “tanya dulu, jawab kemudian”.
  3. Latihan 3: Pecah pesan menjadi dua baris: pertanyaan lalu jawaban, dan pendekkan satu baris.
  4. Latihan 4: Tukar siapa yang bicara dulu dan ubah teksnya.
  5. Latihan 5: Tambahkan orang ketiga dan ganti nama seseorang.

Aturan sintaks PlantUML (daftar lengkap)

  • Mulai dengan @startuml dan akhiri dengan @enduml.
  • Peserta dapat dideklarasikan dengan participant atau dipakai langsung.
  • A -> B adalah panggilan, A --> B adalah balasan.
  • Teks ditulis setelah titik dua: A -> B: Hello.
  • Waktu mengalir dari atas ke bawah.
  • Semakin banyak pesan berarti percakapan lebih panjang.
  • Teks pesan opsional: A -> B tetap valid.
  • Nama yang memakai spasi harus dalam tanda kutip.

Sequence / BASICS / Return Message

Contoh ini menunjukkan “permintaan dan jawaban”, seperti tanya‑jawab.

Mengapa digunakan

  • Menunjukkan langkah permintaan dan jawaban dengan jelas.

Cara menggunakan

  1. A -> B adalah permintaan.
  2. B --> A adalah jawaban.

Contoh penggunaan

  • Panggilan API: Frontend → Backend → Mengembalikan data.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah peran menjadi Client dan Server, dan ubah pesan menjadi “minta data / balas data”.
  2. Latihan 2: Ubah permintaan menjadi “query” dan jawaban menjadi “query result”.
  3. Latihan 3: Tambahkan satu permintaan dan jawaban lagi (empat baris) dan ubah teks kedua.
  4. Latihan 4: Biarkan orang lain yang menjawab dan sesuaikan teksnya.
  5. Latihan 5: Tambahkan self‑message “persiapan” sebelum permintaan.

Aturan sintaks PlantUML (daftar lengkap)

  • -> mengirim pesan.
  • --> sering menunjukkan balasan.
  • Tambahkan teks untuk menjelaskan isi balasan.
  • Permintaan dan jawaban sering berpasangan, tetapi tidak wajib.
  • Deklarasikan peserta dulu jika ingin jelas.
  • Waktu mengalir dari atas ke bawah.

Sequence / BASICS / Self-Message

Ini adalah “bicara dengan diri sendiri”, menunjukkan langkah internal.

Mengapa digunakan

  • Menunjukkan objek yang bekerja di dalam.

Cara menggunakan

  1. Alice -> Alice: Process

Contoh penggunaan

  • Langkah algoritma, perhitungan internal.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ganti Alice dengan namamu (mis. Me) dan ubah teks pesan.
  2. Latihan 2: Ubah aksi menjadi “baca data” dan tambahkan Me -> Me: cek lagi.
  3. Latihan 3: Ubah pesan menjadi “pikirkan” dan tambahkan Me -> Me: tulis poin kunci.
  4. Latihan 4: Ganti nama menjadi Robot dan tambahkan Robot -> Robot: isi ulang.
  5. Latihan 5: Tambahkan Me -> Me: start di awal dan Me -> Me: done di akhir.

Aturan sintaks PlantUML (daftar lengkap)

  • Dua ujung panah bisa peserta yang sama.
  • Format dasar: Role -> Role: Text.
  • Nama boleh dipakai tanpa deklarasi.
  • Teks setelah titik dua bersifat opsional.
  • Lebih banyak self‑message menunjukkan lebih banyak langkah internal.
  • Dapat dipakai bersama activate / deactivate.
  • Waktu mengalir dari atas ke bawah.

Sequence / PARTICIPANTS / Types

Contoh ini menunjukkan ikon berbeda untuk peran berbeda, seperti orang, sistem, basis data.

Mengapa digunakan

  • Ikon membantu mengenali peran lebih cepat.

Cara menggunakan

  1. Gunakan actor, boundary, control, database, entity.

Contoh penggunaan

  • Alur login: User → Web → Logic → Database.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah User menjadi Student dan ubah pesannya.
  2. Latihan 2: Tambahkan boundary App dan arahkan panah ke App dulu.
  3. Latihan 3: Ubah database DB menjadi database Books dan ubah pesannya.
  4. Latihan 4: Tambahkan peran control dan satu baris interaksi.
  5. Latihan 5: Atur ulang peserta dan sesuaikan pesan.

Aturan sintaks PlantUML (daftar lengkap)

  • Tipe peran mengubah tampilan, bukan nama.
  • Gunakan -> untuk interaksi.
  • Format deklarasi: actor/boundary/control/database/entity Name.
  • Deklarasikan peran dulu, lalu gunakan nama.
  • Nama yang tidak dideklarasikan adalah peserta biasa.

Mengapa digunakan

  • Menyorot peran penting dengan warna.

Cara menggunakan

  1. Gunakan skinparam untuk mengatur gaya.
  2. Tambahkan warna setelah peran: participant Alice #AliceBlue.

Contoh penggunaan

  • Menunjukkan “sistem pusat” dengan warna cerah.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah nama Alice dan ganti warnanya.
  2. Latihan 2: Ubah pesan menjadi “start” dan gelapkan peran lain.
  3. Latihan 3: Tambah peran ketiga dengan warna berbeda.
  4. Latihan 4: Ubah skinparam sequenceMessageAlign dan sesuaikan satu baris.
  5. Latihan 5: Gunakan nama gaya (mis. #technology) dan ubah pesan.

Aturan sintaks PlantUML (daftar lengkap)

  • skinparam mengatur perataan, font, warna, dll.
  • Peran bisa diberi warna atau nama gaya.
  • Warna bisa nama atau hex, mis. #AliceBlue.
  • Jika peran dideklarasikan ulang, yang terakhir berlaku.
  • Pengaturan gaya tidak mengubah sintaks pesan.

Sequence / CONTROL FLOW / Alt/Else (Choices)

Menunjukkan “jika… jika tidak…”, seperti pilihan.

Mengapa digunakan

  • Menunjukkan cabang berbeda untuk kondisi berbeda.

Cara menggunakan

  1. Gunakan alt untuk mulai, else untuk cabang lain, end untuk selesai.

Contoh penggunaan

  • Pembayaran berhasil atau gagal.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah kondisi menjadi “punya uang / tidak punya uang” dan ubah pesannya.
  2. Latihan 2: Tambahkan cabang else ketiga dan satu pesan.
  3. Latihan 3: Ganti judul alt dengan skenario kamu.
  4. Latihan 4: Tambahkan satu baris di tiap cabang dan ganti judul.
  5. Latihan 5: Buat cabang “coba lagi setelah error”.

Aturan sintaks PlantUML (daftar lengkap)

  • alt Condition memulai satu cabang.
  • Bisa ada banyak cabang else.
  • Tutup dengan end.
  • Setiap cabang bisa berisi beberapa pesan.
  • Teks kondisi opsional tetapi dianjurkan.

Sequence / CONTROL FLOW / Loop

Menunjukkan aksi yang berulang.

Mengapa digunakan

  • Menunjukkan langkah berulang.

Cara menggunakan

  1. Gunakan loop dan end untuk membungkus baris berulang.

Contoh penggunaan

  • Mengecek status setiap detik.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah label loop menjadi “setiap 3 kali” dan ubah pesannya.
  2. Latihan 2: Tambah pesan lain di dalam loop dan pendekkan yang asli.
  3. Latihan 3: Ubah nama peran dan jadikan pesannya “cek”.
  4. Latihan 4: Ubah label loop menjadi kondisi seperti “hingga selesai”.
  5. Latihan 5: Tambahkan pesan mulai/akhir di luar loop.

Aturan sintaks PlantUML (daftar lengkap)

  • loop count/condition bisa di baris yang sama.
  • end mengakhiri blok loop.
  • Beberapa pesan bisa berada di dalam loop.

Sequence / CONTROL FLOW / Parallel (Par)

Menunjukkan dua hal yang terjadi bersamaan.

Mengapa digunakan

  • Menunjukkan tugas paralel.

Cara menggunakan

  1. Gunakan par untuk mulai, else untuk cabang paralel lain, end untuk selesai.

Contoh penggunaan

  • Memberi tahu dua layanan sekaligus.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah cabang menjadi “kirim email” dan “kirim SMS”, dan ubah perannya.
  2. Latihan 2: Ubah peran dan tambah satu baris di tiap cabang.
  3. Latihan 3: Tambahkan cabang ketiga (else) dan satu pesan.
  4. Latihan 4: Ubah satu cabang jadi “lakukan pelan” dan tambahkan pesan akhir.
  5. Latihan 5: Tambahkan “mulai kerja” sebelum par dan “selesai” setelah end.

Aturan sintaks PlantUML (daftar lengkap)

  • par membuat blok paralel.
  • Setiap cabang memiliki urutan pesan sendiri.
  • else berpindah ke cabang paralel lain.
  • Tutup dengan end.

Sequence / ADVANCED / Activation

Contoh ini memakai batang aktivasi untuk menunjukkan kapan sebuah peran sibuk.

Mengapa digunakan

  • Menunjukkan siapa bekerja dan berapa lama.

Cara menggunakan

  1. Gunakan activate Role untuk mulai.
  2. Gunakan deactivate Role untuk selesai.

Contoh penggunaan

  • Menunjukkan server memproses permintaan.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah peran menjadi Server dan pesan menjadi “processing”.
  2. Latihan 2: Tambah satu baris permintaan dan taruh activate setelahnya.
  3. Latihan 3: Tambah pesan “halfway” dan ganti dengan teksmu.
  4. Latihan 4: Tambah peran lain beserta aktivasi dan pesannya.
  5. Latihan 5: Geser awal/akhir aktivasi satu baris ke bawah.

Aturan sintaks PlantUML (daftar lengkap)

  • activate/deactivate dapat muncul sebelum atau sesudah pesan.
  • activate dan deactivate harus berpasangan.
  • Pesan dapat dikirim ketika peran aktif.

Use Case / BASICS / Simple Use Case

Diagram use case adalah daftar “apa yang bisa dilakukan sistem”.

Mengapa digunakan

  • Mencatat fungsi yang dibutuhkan pengguna secara jelas.

Cara menggunakan

  1. Gunakan actor untuk pengguna.
  2. Gunakan (UseCase) untuk sebuah fungsi.

Contoh penggunaan

  • Login, mencari, pembayaran, dll.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah pengguna menjadi Teacher dan use case menjadi “create quiz”.
  2. Latihan 2: Tambahkan use case kedua.
  3. Latihan 3: Ganti nama use case dan aktor.
  4. Latihan 4: Tambahkan aktor kedua dan hubungkan ke use case.
  5. Latihan 5: Gunakan skenario kamu.

Aturan sintaks PlantUML (daftar lengkap)

  • Use case ditulis dalam tanda kurung: (Login).
  • Aktor adalah gambar orang: actor User.
  • Hubungkan aktor ke use case dengan garis.

Use Case / RELATIONSHIPS / Include/Extend

Menunjukkan hubungan include/extend antar use case.

Mengapa digunakan

  • Menandai langkah yang dibagi atau opsional.

Cara menggunakan

  1. <> berarti “selalu disertakan”.
  2. <> berarti “opsional”.

Contoh penggunaan

  • Login selalu menyertakan validasi; pembayaran bisa memperluas diskon.

Latihan (mudah untuk anak-anak)

  1. Latihan 1: Ubah use case dengan contohmu.
  2. Latihan 2: Tukar include dan extend.
  3. Latihan 3: Tambah use case lain dan include-kan.
  4. Latihan 4: Balik arah panah dan perbarui teks.
  5. Latihan 5: Ubah teks relasi dengan kata-katamu.

Aturan sintaks PlantUML (daftar lengkap)

  • Sintaks include: (A) .> (B) : <>.
  • Sintaks extend: (A) .> (B) : <>.
  • Gunakan panah garis putus-putus.

  1. Definisikan kelas dengan class.
  2. Letakkan atribut dan metode di dalam kurung kurawal.
