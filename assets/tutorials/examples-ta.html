Sequence / BASICS / Simple Message

இந்த உதாரணம் “நபர் A, நபர் B‑க்கு ஒரு செய்தி அனுப்புகிறார்” என்பதைக் காட்டுகிறது; ஒரு குறிப்பு சீட்டு கொடுப்பதைப் போல.

ஏன் பயன்படுத்த வேண்டும்

  • உரையாடலின் வரிசையை எளிதாகப் புரியுமாறு காட்டுகிறது.
  • “யார் முதலில் பேசுகிறார், யார் பிறகு பதில் தருகிறார்” போன்ற கதைகளுக்கு சிறந்தது.

எப்படி பயன்படுத்துவது

  1. @startuml மற்றும் @enduml எழுதுங்கள்.
  2. Alice -> Bob: Message மூலம் A, B‑யுடன் பேசுவதை காட்டுங்கள்.

எடுத்துக்காட்டான பயன்பாடுகள்

  • லாகின் ஓட்டம்: User → System → Server.
  • சாட்போட்: நீங்கள் கேட்கிறீர்கள் → பாட்டு பதில் தருகிறது.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: Alice மற்றும் Bob ஐ விருப்பமான பெயர்களாக மாற்றி, செய்தியை தினசரி உரையாடலாக மாற்றுங்கள்.
  2. பயிற்சி 2: முதலில் participant வரியை சேர்த்து, பின்னர் “முதலில் கேள், பிறகு பதில்” என்று மாற்றுங்கள்.
  3. பயிற்சி 3: செய்தியை இரண்டு வரிகளாகப் பிரிக்கவும்: முதலில் கேள், பிறகு பதில்; ஒரு வரியை சுருக்கவும்.
  4. பயிற்சி 4: முதலில் பேசுபவரை மாற்றி, செய்தி உரையையும் மாற்றுங்கள்.
  5. பயிற்சி 5: மூன்றாவது நபரை சேர்த்து ஒரு வரி பேசச் செய்யுங்கள்; ஒரு பெயரை மாற்றவும்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • @startuml இல் தொடங்கி @enduml இல் முடிக்கவும்.
  • பங்கேற்பாளர்களை participant மூலம் அறிவிக்கலாம் அல்லது நேரடியாக பயன்படுத்தலாம்.
  • A -> B என்பது அழைப்பு, A --> B என்பது பதில்.
  • கோலன் பிறகு உரை எழுதவும்: A -> B: Hello.
  • நேரம் மேல் முதல் கீழ் செல்கிறது.
  • அதிக செய்தி வரிகள் நீளமான உரையாடலைக் காட்டும்.
  • செய்தி உரை விருப்பமானது: A -> B செல்லுபடியாகும்.
  • இடைவெளியுள்ள பெயர்கள் quotes உள்ளே இருக்க வேண்டும்.

Sequence / BASICS / Return Message

இந்த உதாரணம் “கோரிக்கை மற்றும் பதில்” என்பதை காட்டுகிறது, கேள்வி‑பதில் போல.

ஏன் பயன்படுத்த வேண்டும்

  • கோரிக்கை மற்றும் பதில் படிகளை தெளிவாக காட்டுகிறது.

எப்படி பயன்படுத்துவது

  1. A -> B கோரிக்கை.
  2. B --> A பதில்.

எடுத்துக்காட்டான பயன்பாடுகள்

  • API அழைப்பு: Frontend → Backend → தரவு திரும்புகிறது.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: பாத்திரங்களை Client மற்றும் Server ஆக மாற்றி, செய்தியை “data கேள் / data திருப்பி அனுப்பு” என்று மாற்றுங்கள்.
  2. பயிற்சி 2: கோரிக்கையை “query” என்றும் பதிலை “query result” என்றும் மாற்றுங்கள்.
  3. பயிற்சி 3: இன்னொரு கோரிக்கை‑பதில் ஜோடியை சேர்த்து (மொத்தம் 4 வரிகள்), இரண்டாவது உரையை மாற்றுங்கள்.
  4. பயிற்சி 4: வேறொருவரை பதில் சொல்லச் செய்து, உரையை சரி செய்யுங்கள்.
  5. பயிற்சி 5: கோரிக்கைக்கு முன் “preparing” self‑message ஒன்றைச் சேர்த்து, பின்னர் கோரிக்கையை தெளிவாக்குங்கள்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • -> ஒரு செய்தியை அனுப்பும்.
  • --> பொதுவாக பதிலை காட்டும்.
  • பதிலின் பொருளை விளக்க உரை சேர்க்கவும்.
  • கோரிக்கை‑பதில் ஜோடி பொதுவானது; கட்டாயமில்லை.
  • தெளிவுக்கு பங்கேற்பாளர்களை முன்பே அறிவிக்கலாம்.
  • நேரம் மேலிருந்து கீழே செல்லும்.

Sequence / BASICS / Self-Message

இது “தன்னுடன் பேசுவது,” ஒரு உள்ளக படியை காட்டுகிறது.

ஏன் பயன்படுத்த வேண்டும்

  • ஒரு பொருளின் உள்ளக பணியை மற்றவர்கள் இல்லாமல் காட்டுகிறது.

எப்படி பயன்படுத்துவது

  1. Alice -> Alice: Process

எடுத்துக்காட்டான பயன்பாடுகள்

  • அல்காரிதம் படிகள், உள்ளக கணக்குகள்.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: Alice ஐ உங்கள் பெயராக (எ.கா. Me) மாற்றி, இன்று செய்ய விரும்பும் செயலை எழுதுங்கள்.
  2. பயிற்சி 2: செயலை “தரவு படி” என மாற்றி, Me -> Me: மீண்டும் சரிபார் சேர்க்கவும்.
  3. பயிற்சி 3: செய்தியை “சிந்தி” என்று மாற்றி, Me -> Me: முக்கிய குறிப்புகள் எழுத சேர்க்கவும்.
  4. பயிற்சி 4: பெயரை Robot என்று மாற்றி, Robot -> Robot: சார்ஜ் சேர்க்கவும்.
  5. பயிற்சி 5: மேலே Me -> Me: start மற்றும் கீழே Me -> Me: done சேர்க்கவும்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • அம்பின் இரு முனைகளும் ஒரே பங்கேற்பாளராக இருக்கலாம்.
  • அடிப்படை வடிவம்: Role -> Role: Text.
  • முன் அறிவிப்பு இல்லாமல் பெயர்களை பயன்படுத்தலாம்.
  • கோலன் பிறகு உரை விருப்பமானது.
  • பல self‑message கள் பல உள்ளக படிகளை காட்டும்.
  • activate / deactivate உடன் பயன்படுத்தலாம்.
  • நேரம் மேல் முதல் கீழ்.

Sequence / CONTROL FLOW / Alt/Else (Choices)

இந்த உதாரணம் “என்றால்… இல்லையெனில்…” போன்ற தேர்வுகளை காட்டுகிறது.

ஏன் பயன்படுத்த வேண்டும்

  • வேறு நிபந்தனைகளுக்கு வேறு கிளைகள் காட்டும்.

எப்படி பயன்படுத்துவது

  1. alt கொண்டு தொடங்கி, மற்ற கிளைக்கு else, முடிக்க end.

எடுத்துக்காட்டான பயன்பாடுகள்

  • கட்டணம் வெற்றி அல்லது தோல்வி.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: நிபந்தனைகளை “பணம் உள்ளது / இல்லை” ஆக மாற்றி செய்தியை மாற்றுங்கள்.
  2. பயிற்சி 2: மூன்றாவது else கிளை சேர்த்து ஒரு செய்தி எழுதுங்கள்.
  3. பயிற்சி 3: alt தலைப்பை உங்கள் சூழலாக மாற்றி செய்திகளை திருத்துங்கள்.
  4. பயிற்சி 4: ஒவ்வொரு கிளைக்கும் ஒரு வரி சேர்த்து தலைப்புகளை மாற்றுங்கள்.
  5. பயிற்சி 5: “தோல்விக்குப் பிறகு மீண்டும் முயற்சி” கிளை உருவாக்குங்கள்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • alt Condition ஒரு கிளையை தொடங்கும்.
  • பல else கிளைகள் இருக்கலாம்.
  • end மூலம் முடிக்கவும்.
  • ஒவ்வொரு கிளையிலும் பல செய்திகள் இருக்கலாம்.
  • நிபந்தனை உரை விருப்பமானது ஆனால் பரிந்துரை செய்யப்படுகிறது.

Sequence / CONTROL FLOW / Loop

மறுமுறை செய்யும் செயலை காட்டுகிறது.

ஏன் பயன்படுத்த வேண்டும்

  • மீண்டும் செய்யும் படிகளை காட்டுகிறது.

எப்படி பயன்படுத்துவது

  1. மீண்டும் வரும் வரிகளை loop மற்றும் end குள் வையுங்கள்.

எடுத்துக்காட்டான பயன்பாடுகள்

  • ஒவ்வொரு நொடியும் நிலையைச் சோதித்தல்.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: loop லேபிளை “ஒவ்வொரு 3 முறை” என்று மாற்றி செய்தியை மாற்றுங்கள்.
  2. பயிற்சி 2: loop உள்ளே ஒரு செய்தி சேர்த்து, முதன்மை செய்தியை சுருக்குங்கள்.
  3. பயிற்சி 3: பங்குகளை மாற்றி செய்தியை “சோதிக்கிறது” என்று மாற்றுங்கள்.
  4. பயிற்சி 4: loop லேபிளை “முடியும் வரை” போன்ற நிபந்தனையாக்குங்கள்.
  5. பயிற்சி 5: loop வெளியே தொடக்க/முடிவு செய்திகள் சேர்க்கவும்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • loop count/condition ஒரே வரியில் இருக்கலாம்.
  • end loop block ஐ முடிக்கும்.
  • loop இல் பல செய்திகள் இருக்கலாம்.

Sequence / CONTROL FLOW / Parallel (Par)

ஒரே நேரத்தில் இரண்டு செயல்கள் நடப்பதை காட்டுகிறது.

ஏன் பயன்படுத்த வேண்டும்

  • இணைச்செயல்களை காட்டும்.

எப்படி பயன்படுத்துவது

  1. par தொடங்கி, மற்ற இணை கிளைக்காக else, முடிக்க end.

எடுத்துக்காட்டான பயன்பாடுகள்

  • ஒரே நேரத்தில் இரண்டு சேவைகளுக்கு அறிவிப்பு.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: கிளைகளை “email அனுப்பு” மற்றும் “SMS அனுப்பு” என மாற்றி பங்குகளை மாற்றுங்கள்.
  2. பயிற்சி 2: பங்குகளை மாற்றி ஒவ்வொரு கிளைக்கும் ஒரு வரி சேர்க்கவும்.
  3. பயிற்சி 3: மூன்றாவது கிளை (else) சேர்த்து ஒரு செய்தி எழுதுங்கள்; ஒரு வரியை சுருக்கவும்.
  4. பயிற்சி 4: ஒரு கிளையை “மெதுவாக செய்” என்று மாற்றி, முடிவு செய்தி சேர்க்கவும்.
  5. பயிற்சி 5: par க்கு முன் “பணி தொடங்கு” மற்றும் end பின் “முடிந்தது” சேர்க்கவும்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • par இணை block உருவாக்கும்.
  • ஒவ்வொரு கிளைக்கும் தனி செய்தி வரிசை இருக்கலாம்.
  • else மற்ற இணை கிளைக்கு மாற்றும்.
  • end

Sequence / ADVANCED / Activation

இங்கு activation bars மூலம் ஒரு பங்கு எப்போது பிஸியாக இருக்கிறது என்பதைக் காட்டுகிறது.

ஏன் பயன்படுத்த வேண்டும்

  • யார் வேலை செய்கிறார், எவ்வளவு நேரம் என்பதை காட்டும்.

எப்படி பயன்படுத்துவது

  1. தொடங்க activate Role.
  2. முடிக்க deactivate Role.

எடுத்துக்காட்டான பயன்பாடுகள்

  • சர்வர் செயலாக்க நேரத்தை காட்டுதல்.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: பங்கை Server ஆக மாற்றி “processing” என்று மாற்றுங்கள்.
  2. பயிற்சி 2: ஒரு கோரிக்கை வரி சேர்த்து அதற்குப் பிறகு activate வை இடுங்கள்.
  3. பயிற்சி 3: “halfway” போன்ற நடுப்பாதி செய்தியை சேர்த்து உங்கள் உரையால் மாற்றுங்கள்.
  4. பயிற்சி 4: இன்னொரு பங்கை அதன் activation மற்றும் செய்தியுடன் சேர்க்கவும்.
  5. பயிற்சி 5: activation தொடக்க/முடிவை ஒரு வரி கீழே நகர்த்துங்கள்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • activate/deactivate செய்திக்கு முன் அல்லது பின் இருக்கலாம்.
  • activate மற்றும் deactivate ஜோடியாக இருக்க வேண்டும்.
  • பங்கு active ஆக இருக்கும் போது செய்திகள் சேர்க்கலாம்.

Use Case / BASICS / Simple Use Case

Use case வரைபடம் “அமைப்பு செய்யக்கூடியவை” என்ற பட்டியல்.

ஏன் பயன்படுத்த வேண்டும்

  • பயனாளி தேவைப்படும் செயல்பாடுகளை தெளிவாக காட்டும்.

எப்படி பயன்படுத்துவது

  1. பயனாளிக்காக actor பயன்படுத்தவும்.
  2. செயல்பாட்டுக்கு (UseCase) பயன்படுத்தவும்.

எடுத்துக்காட்டான பயன்பாடுகள்

  • லாகின், தேடல், கட்டணம் முதலியவை.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: பயனாளியை Teacher ஆக மாற்றி, use case ஐ “create quiz” ஆக மாற்றுங்கள்.
  2. பயிற்சி 2: இன்னொரு use case சேர்க்கவும்.
  3. பயிற்சி 3: use case மற்றும் actor பெயர்களை மாற்றவும்.
  4. பயிற்சி 4: இரண்டாவது actor சேர்த்து ஒரு use case க்கு இணைக்கவும்.
  5. பயிற்சி 5: உங்கள் சொந்த சூழலைப் பயன்படுத்தவும்.

  • Use case வட்டக்குறியீட்டில் எழுதப்படும்: (Login).
  • Actor ஒரு stick figure: actor User.
  • Actor ஐ use case க்கு கோடால் இணைக்கவும்.

Use Case / RELATIONSHIPS / Include/Extend

Use case களுக்கிடையிலான include/extend தொடர்பை காட்டுகிறது.

ஏன் பயன்படுத்த வேண்டும்

  • பொது படிகள் அல்லது விருப்ப ஓட்டங்களை குறிக்கிறது.

எப்படி பயன்படுத்துவது

  1. <> என்பது “எப்போதும் சேரும்”.
  2. <> என்பது “விருப்பம்”.

எடுத்துக்காட்டான பயன்பாடுகள்

  • லாகின் கட்டாய சரிபார்ப்பை கொண்டது; கட்டணம் தள்ளுபடியை extend ஆகக் கொண்டிருக்கலாம்.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: Use case பெயர்களை உங்கள் உதாரணமாக மாற்றுங்கள்.
  2. பயிற்சி 2: include மற்றும் extend ஐ மாற்றி இடுங்கள்.
  3. பயிற்சி 3: புதிய use case சேர்த்து include செய்யுங்கள்.
  4. பயிற்சி 4: அம்பை மாறாக மாற்றி உரையை புதுப்பிக்கவும்.
  5. பயிற்சி 5: தொடர்பின் உரையை உங்கள் சொற்களாக மாற்றுங்கள்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • Include syntax: (A) .> (B) : <>.
  • Extend syntax: (A) .> (B) : <>.
  • துளை கோடான அம்பை பயன்படுத்துங்கள்.

எப்படி பயன்படுத்துவது

  • பிரோகிராமின் data model வடிவமைப்பு.

பயிற்சி (குழந்தைகளுக்கு எளிது)

Class / BASICS / Methods

இந்த உதாரணம் methods மற்றும் return types சேர்க்கிறது.

எப்படி பயன்படுத்துவது

  1. Methods வடிவம்: name(): type.

எடுத்துக்காட்டான பயன்பாடுகள்

பயிற்சி (குழந்தைகளுக்கு எளிது)

Class / RELATIONSHIPS / Inheritance

  4. பயிற்சி 4: புதிய child class ஒன்றைச் சேர்க்கவும்.
  5. பயிற்சி 5: உங்கள் சூழலை பயன்படுத்துங்கள்.

  • Child --|> Parent என்பது inheritance.
  • அம்பு parent க்கு சுட்டும்.

Class / RELATIONSHIPS / Composition

இந்த உதாரணம் composition (மிக வலுவான உரிமை) காட்டுகிறது.

ஏன் பயன்படுத்த வேண்டும்

  • முழுதில்லாமல் பகுதி இருக்க முடியாது என்பதைக் காட்டுகிறது.

எப்படி பயன்படுத்துவது

  1. composition க்கு *-- பயன்படுத்துங்கள்.

எடுத்துக்காட்டான பயன்பாடுகள்

  • ஒரு வீடு எப்போதும் அறைகள் கொண்டுள்ளது.

பயிற்சி (குழந்தைகளுக்கு எளிது)

  1. பயிற்சி 1: உங்கள் உதாரணமாக மாற்றுங்கள்.
  2. பயிற்சி 2: composition ஐ aggregation ஆக மாற்றி ஒப்பிடுங்கள்.
  3. பயிற்சி 3: இன்னொரு composition வரியை சேர்க்கவும்.
  4. பயிற்சி 4: உறவு லேபிள் உரையை மாற்றுங்கள்.
  5. பயிற்சி 5: திசையை மாற்றி பாருங்கள்.

PlantUML விதிகள் (முழுப் பட்டியல்)

  • Whole *-- Part என்பது composition.
  • கட்டியான வைரம் உரிமையாளர் பக்கம் இருக்கும்.
