Sequence / BASICS / Simple Message

یہ مثال دکھاتی ہے “شخص A، شخص B کو پیغام بھیجتا ہے”، جیسے کوئی پرچی دینا۔

کیوں استعمال کریں

  • بات چیت کی ترتیب سادہ انداز میں دکھاتی ہے۔
  • “کون پہلے بولتا ہے، کون بعد میں جواب دیتا ہے” جیسی کہانیوں کے لیے بہترین۔

کیسے استعمال کریں

  1. @startuml اور @enduml لکھیں۔
  2. Alice -> Bob: Message سے A کا B کو کہنا دکھائیں۔

ممکنہ استعمالات

  • لاگ اِن فلو: User → System → Server.
  • چیٹ بوٹ: آپ سوال کریں → بوٹ جواب دے۔

مشقیں (بچوں کے لیے آسان)

  1. مشق 1: Alice اور Bob کو پسندیدہ ناموں سے بدلیں، اور جملہ روزمرہ گفتگو بنائیں۔
  2. مشق 2: پہلے participant کی لائن شامل کریں، پھر “پہلے سوال، بعد میں جواب” کریں۔
  3. مشق 3: پیغام کو دو لائنوں میں تقسیم کریں: سوال پھر جواب، اور ایک لائن کو مختصر کریں۔
  4. مشق 4: پہلے بولنے والے کو بدلیں اور متن تبدیل کریں۔
  5. مشق 5: تیسرا شخص شامل کریں اور کسی کا نام بدلیں۔

PlantUML کے قواعد (مکمل فہرست)

  • @startuml سے شروع کریں اور @enduml پر ختم کریں۔
  • شرکاء کو participant سے ظاہر کیا جا سکتا ہے یا سیدھا استعمال کیا جا سکتا ہے۔
  • A -> B کال ہے، A --> B جواب ہے۔
  • متن کولن کے بعد لکھا جاتا ہے: A -> B: Hello.
  • وقت اوپر سے نیچے چلتا ہے۔
  • زیادہ پیغامات لمبی گفتگو ظاہر کرتے ہیں۔
  • پیغام کا متن اختیاری ہے: A -> B درست ہے۔
  • اسپیس والے نام کو کوٹس میں لکھیں۔

Sequence / BASICS / Return Message

یہ مثال “درخواست اور جواب” دکھاتی ہے، جیسے سوال و جواب۔

کیوں استعمال کریں

  • درخواست اور جواب کے مراحل واضح کرتی ہے۔

کیسے استعمال کریں

  1. A -> B درخواست ہے۔
  2. B --> A جواب ہے۔

ممکنہ استعمالات

  • API کال: Frontend → Backend → ڈیٹا واپس۔

مشقیں (بچوں کے لیے آسان)

  1. مشق 1: کرداروں کو Client اور Server کریں، پیغام “data طلب / data واپس” کر دیں۔
  2. مشق 2: درخواست کو “query” اور جواب کو “query result” کریں۔
  3. مشق 3: ایک اور درخواست‑جواب جوڑا شامل کریں (4 لائنیں) اور دوسری لائن کا متن بدلیں۔
  4. مشق 4: کسی اور کو جواب دینے دیں اور متن ایڈجسٹ کریں۔
  5. مشق 5: درخواست سے پہلے “تیاری” والا self‑message شامل کریں۔

PlantUML کے قواعد (مکمل فہرست)

  • -> پیغام بھیجتا ہے۔
  • --> عموماً جواب دکھاتا ہے۔
  • جواب کا مطلب سمجھانے کے لیے متن لکھیں۔
  • درخواست اور جواب اکثر جوڑی میں ہوتے ہیں مگر لازمی نہیں۔
  • واضحی کے لیے پہلے شرکاء ظاہر کر سکتے ہیں۔
  • وقت اوپر سے نیچے جاتا ہے۔

  • بنیادی فارم: Role -> Role: Text.
  • نام بغیر اعلان کے بھی استعمال ہو سکتے ہیں۔
  • کولن کے بعد کا متن اختیاری ہے۔
  • ایک سے زیادہ self‑message مزید اندرونی مراحل دکھاتے ہیں۔
  • activate / deactivate کے ساتھ استعمال ممکن ہے۔
  • وقت اوپر سے نیچے چلتا ہے۔
